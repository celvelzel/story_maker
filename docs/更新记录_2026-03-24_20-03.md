# 更新记录

**日期：** 2026年03月24日 20:03

## 变更摘要

本次更新为 StoryWeaver 增加了可视化“加载存档”入口，并在自动快照保存时调用 LLM 生成短标题，方便用户区分不同存档。同步加强了存档恢复流程的异常安全与状态恢复完整性。

### 功能变更

- 新增 **Save / Load** 侧边栏入口：用户可直接读取历史 `.json` 存档。
- 新增快照标题能力：回合快照保存时自动写入 `save_title`，并在 UI 列表中展示。
- 优化存档加载安全性：加载失败时返回错误信息，避免会话状态被部分污染。
- 优化存档兼容性：对异常 `turn_id` 数据提供安全回退解析。
- 优化恢复一致性：存档恢复时完整恢复冲突策略、抽取策略、重要性策略、摘要策略及冲突统计。

### 技术细节

- 采用 `StoryGenerator.generate_save_title()` 复用现有 LLM 客户端流程，限制输出长度并提供 fallback (`Untitled Journey`)。
- `GameEngine._auto_save()` 在快照分支收集最近叙事上下文，调用标题生成后写入快照文件。
- `app.py` 新增 `_list_save_slots()` / `_restore_from_save_file()`，并采用“先计算后写会话”的原子恢复策略。
