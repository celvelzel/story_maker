# 本地模型推理集成 - 2026-03-27 21:07

**日期：** 2026年03月27日 21:07

## 变更摘要

新增本地 llama.cpp 推理支持，项目可通过本地 CPU 运行 Qwen3-4B 量化模型，无需依赖远程 API 或 GPU，同时保持与现有 OpenAI 兼容 API 的无缝切换。

## 修改详情

### 新增文件

| 文件路径 | 修改类型 | 说明 |
|---------|---------|------|
| scripts/start_llama_server.bat | 新增 | 本地 llama.cpp 服务器启动脚本 |
| .env.llama | 新增 | 本地模型配置参考文件 |
| .sisyphus/plans/ui-model-toggle.md | 新增 | UI 切换功能实现计划 |

### 修改文件

| 文件路径 | 修改类型 | 说明 |
|---------|---------|------|
| .env | 修改 | 更新为三组后端选项（远程/vLLM/本地），默认激活本地 llama.cpp |

### 功能变更

- 新增本地 llama.cpp 服务器启动脚本 `scripts/start_llama_server.bat`
  - 支持配置：端口、线程数、上下文长度、批处理大小
  - 自动检测模型文件和服务器二进制文件
  - 启动 OpenAI 兼容 API（端口 8081）
- 更新 `.env` 配置文件
  - 方案 A：远程 Mimo API（默认注释）
  - 方案 B：vLLM 本地 GPU 推理（默认注释）
  - 方案 C：llama.cpp 本地 CPU 推理（默认激活）
- 创建 UI 切换功能实现计划
  - 侧边栏后端选择器
  - 动态切换 LLM 客户端
  - 服务器健康状态检测

### 技术细节

- **模型转换**：HuggingFace 格式 → GGUF（FP16）→ 量化（Q4_K_M）
- **量化效果**：7.5GB（FP16）→ 2.4GB（Q4_K_M），压缩率 68%
- **服务器**：llama-server b5200 版本，OpenAI 兼容 API
- **端点**：
  - Chat: `http://127.0.0.1:8081/v1/chat/completions`
  - Models: `http://127.0.0.1:8081/v1/models`
- **配置**：现有 `api_client.py` 无需修改，通过 `OPENAI_BASE_URL` 支持任意兼容服务
