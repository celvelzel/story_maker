## 1. 变更摘要

本次更新为 StoryWeaver 引入了可视化“加载存档”功能，并实现了基于 LLM 的自动快照命名。用户现在可以更方便地管理与区分不同的游戏存档。此外，更新还显著增强了存档恢复流程的异常安全性与系统状态的完整性。

## 2. 修改详情

### 2.1 关键文件变更

| 模块 | 文件路径 | 变更说明 |
|---------|---------|------|
| **前端** | `app.py` | 侧边栏新增存档管理面板（列表展示/选择/一键加载），强化异常保护逻辑 |
| **引擎** | `src/engine/game_engine.py` | 存档格式升级，新增 `save_title` 与 `turn_conflict_counts` 持久化字段 |
| **生成器** | `src/nlg/story_generator.py` | 新增 `generate_save_title()`，通过 LLM 为存档生成叙事化短标题 |
| **提示词** | `src/nlg/prompt_templates.py` | 定义 `SAVE_TITLE_PROMPT`，引导模型生成简短、有代表性的存档名称 |
| **测试** | `tests/engine/*.py` | 新增快照标题持久化与复杂策略状态恢复的专项测试 |

### 2.2 功能增强

- **可视化 Save / Load**: 在 Streamlit 侧边栏提供了直观的存档管理入口，支持读取所有 `.json` 格式的游戏快照。
- **智能快照命名**: 系统在自动保存快照时，会根据最近的叙事上下文调用 LLM 生成一个具有代表性的短标题（如“幽暗地穴的转机”），并在加载列表中展示。
- **原子化恢复策略**: 存档加载采用“先计算/校验、后写入会话”的策略。若加载失败，会立即拦截错误并保持当前游戏状态不被污染。
- **策略一致性恢复**: 存档不仅恢复剧情，还会完整恢复存档时的 KG 冲突策略、抽取模式、重要性算法及摘要格式。

## 3. 技术规范

- **标题生成 Fallback**: 限制标题输出长度，若 LLM 生成失败，系统将自动回退至默认标题 (`Untitled Journey`)。
- **自动保存触发**: `GameEngine._auto_save()` 在特定回合触发快照分支，异步（或阻塞式，取决于配置）收集上下文并调用生成接口。
- **UI 列表解析**: `app.py` 中的 `_list_save_slots()` 负责扫描 `saves/` 目录并提取元数据，支持按时间排序。
